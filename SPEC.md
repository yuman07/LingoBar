# LingoBar

## 一、产品概述

LingoBar 是一款 macOS 菜单栏翻译工具。它常驻系统菜单栏，用户点击图标或按下全局快捷键即可唤出翻译窗口，快速完成多语言翻译。

产品定位：轻量、快速、原生。不做大而全的翻译平台，只做"点一下就能翻译"的效率工具。纯 Swift + AppKit 实现，无 WebView，遵循 Apple 设计语言，追求原生体验。

默认使用 Apple 原生 Translation 框架提供翻译能力，开箱即用，无需配置。用户可在设置中自行添加 Google 翻译（走其公开网页端点，无需 API Key、无需登录）及后续可能新增的引擎，并自由排序。

---

## 二、产品目标

1. 从唤出翻译窗口到看到翻译结果，整个流程在 2 秒内完成（网络正常时）
2. 支持至少 20 种常见语言的互译（取各翻译引擎支持语言的交集）
3. 支持 2 个翻译引擎：Apple 翻译（默认，免费，离线）、Google 翻译（走公开网页端点，免费，无需配置）
4. 内存占用常驻状态下不超过 50MB
5. UI 简洁美观，符合 macOS 原生设计语言

---

## 三、目标用户

1. **多语言工作者**：日常需要频繁查词、翻译短句的开发者、设计师、产品经理等，核心诉求是快速获取翻译结果，不打断工作流
2. **语言学习者**：需要随时查询词义、对比不同翻译引擎结果的用户，核心诉求是方便对照和朗读

## 五、核心用户流程

### 第一步：首次启动

1. 用户首次打开 App，菜单栏出现 LingoBar 图标
2. App 首次启动引擎列表仅含 Apple 翻译引擎，开箱即用，无需任何配置
3. 如果 Apple 翻译需要下载语言包，自动提示用户下载

> **规则**：App 首次启动即可翻译，无需任何前置配置。用户可在设置的"翻译引擎"页自行添加 Google 及其他支持的引擎，并调整优先级顺序

### 第二步：唤出翻译窗口

用户通过以下任一方式唤出翻译窗口：
- **点击菜单栏图标**：翻译窗口从菜单栏图标下方以 Popover 形式弹出
- **全局快捷键**：按下快捷键（默认 ⌥⌘T），翻译窗口同样从菜单栏图标下方弹出，效果与点击菜单栏图标完全一致

> **规则**：两种唤出方式共用同一个窗口实例；窗口已显示时再次触发，关闭窗口（toggle 行为）

### 第三步：输入与翻译

1. 翻译窗口弹出后，输入框自动获得焦点
2. 用户在输入框中输入文本
3. 实时翻译：用户停止输入约 500ms 后自动触发翻译
4. 翻译结果显示在输出框中

> **语言选择规则**：
> - 输入语言：默认"自动检测"，用户可手动切换为指定语言
> - 输出语言：选择器包含"自动"选项和所有具体语言。"自动"模式下启用智能切换（检测到中文→输出英文，其他语言→输出中文）；选择具体语言时固定为该语言
> - 输出语言默认值："自动"
> - 语言选择记忆：输入语言和输出语言的选择在 App 重启后保持

### 第四步：结果操作

用户可对输入文本和翻译结果执行以下操作：
- **复制**：点击复制按钮，将文本复制到剪贴板
- **朗读**：点击朗读按钮，使用系统 TTS 朗读文本

### 第五步：关闭窗口

用户通过以下任一方式关闭翻译窗口：
- 点击窗口外部区域
- 按 Esc 键
- 再次点击菜单栏图标
- 再次按下全局快捷键

> **内容保留规则**：关闭窗口后开始计时，3 分钟内重新打开仍保留上次输入内容、翻译结果和所在 Tab；超过 3 分钟则清空输入和输出，并将下次打开时的 Tab 重置为"翻译"。保留时长固定，不在设置中暴露

> **锁定保持规则**：翻译窗口输入区右侧提供锁定按钮。锁定后，窗口不再因点击外部区域或按 Esc 键而自动关闭，仅通过菜单栏图标（Popover）或全局快捷键（NSPanel）的显式 toggle 才会关闭；解锁后恢复默认关闭行为。锁定状态不跨 App 重启保持

## 六、功能模块详述

### 1. 翻译窗口 UI

窗口采用上下分区布局，固定尺寸，不可调整大小。结构如下：

```
┌──────────────────────────┐
│ [翻译] [历史] [设置]     │  ← 顶部 Tab 切换
├──────────────────────────┤
│ [自动检测 ▼]             │  ← 输入语言选择器
│                          │
│  输入文本区        [📋][🔊]│  ← 输入框 + 复制/朗读按钮
│                          │
├──────────────────────────┤
│ [自动 ▼]        [🍎Apple]│  ← 输出语言 + 引擎来源标识
│                          │
│  翻译结果区        [📋][🔊]│  ← 输出框 + 复制/朗读按钮
│                          │
└──────────────────────────┘
```

- **窗口类型**：统一以 Popover 形式从菜单栏图标下方弹出，无论由点击图标还是全局快捷键触发
- **引擎来源标识**：输出区域顶部右侧显示当前翻译结果的来源引擎图标 + 名称（如 Apple 图标、Google 图标等），让用户一目了然翻译来自哪个引擎。引擎自动切换时，标识同步更新为实际提供结果的引擎
- **输入/输出框**：支持多行文本，内容超出时可滚动
- **历史视图**：切换到"历史"Tab 后，显示历史记录列表，顶部有搜索框，每条记录显示原文摘要、译文摘要、引擎图标和时间
- **设置视图**：切换到"设置"Tab 后，在翻译窗口内以可滚动表单形式展示所有配置项，无独立设置窗口

### 2. 翻译引擎管理

App 当前支持以下翻译引擎，用户按需启用并排序：

| 引擎 | 类型 | 配置项 | 备注 |
|------|------|--------|------|
| Apple 翻译 | 内置免费 | 无需配置 | macOS Translation 框架，支持离线 |
| Google 翻译 | 内置免费 | 无需配置 | 走 `translate.googleapis.com/translate_a/single?client=gtx` 公开网页端点，无需 API Key / 登录 / 绑卡 |

**启用与排序**：设置中的"翻译引擎"子页以有序列表维护当前启用的引擎。首次安装列表只包含 Apple；用户可添加任意"未在列表中"的受支持引擎（同一引擎不重复），也可删除任意引擎（包括 Apple）；支持拖拽调整顺序。列表至少保留 1 个引擎，不允许清空

**调用顺序**：翻译时按列表从上到下依次尝试。列表首位即"当前活动引擎"。输入框在任意时刻变为空（首次打开、手动清空、3 分钟过期清除等）都会把活动引擎重置为列表首位；新的翻译先调用活动引擎，若失败（错误或超时）则继续往下尝试，直到某一个返回结果或全部失败。最终成功的引擎即成为当前活动引擎，其图标显示在输出区的引擎标签上

**统一超时**：所有引擎共用同一个"单次请求超时时间"，默认 5 秒，可在设置中调整，最小 1 秒。超过该时间未返回结果视为失败并切到下一个引擎

**兜底策略**：无隐藏兜底。引擎全部失败即展示错误信息；若用户已删除 Apple，则不再偷偷回退到 Apple

**实时响应**：在设置中调整列表、顺序、删除、新增或修改超时后，若翻译窗口正在显示，自动用新配置重新翻译

### 3. 翻译历史

- 保存翻译历史记录，包含：原文、译文、源语言、目标语言、使用的引擎、时间戳
- 历史按"输入会话"聚合：从输入框空白到下一次变空白视为一个会话，一个会话只生成一条历史记录；会话内的后续翻译（继续输入、切换语言 / 引擎等）就地更新该条记录的原文、译文、引擎和时间戳，而不是新增条目。清空输入、清除按钮、点击历史条目回填、在历史页删除记录均会结束当前会话，下次翻译重新建立新记录
- 翻译窗口顶部设 Tab 切换（翻译 / 历史 / 设置），在窗口内查看历史
- 历史记录条数上限：500 条，超出后自动删除最早的记录（收藏记录不计入上限、不会被淘汰）
- 支持搜索历史记录（按原文和译文搜索）
- 支持删除单条 / 清空全部历史；"清空全部"不会删除收藏记录，只能通过单条删除按钮移除
- 点击历史记录条目可将内容回填到翻译视图
- 支持收藏单条历史（星标，图标为 `star` / `star.fill`，与翻译 Tab 的"钉住窗口"pin 功能完全区分）：每条记录行在删除按钮左侧提供星标开关，收藏后该记录在下次打开历史 Tab（或历史列表因新增记录等原因刷新）时浮到列表顶部；收藏动作本身不立即触发重排，仅更新星标图标。多条收藏时按收藏时间升序排列（先收藏的靠上、后收藏的靠下），收藏区域下方为未收藏记录的时间倒序

### 4. 设置

设置作为翻译窗口内第三个 Tab（翻译、历史之后），不再以独立窗口打开。右键菜单"设置…"项直接唤起翻译窗口并切换到该 Tab。设置页面顶层包含以下配置项：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 翻译引擎 | 进入引擎子页管理列表、顺序与超时 | 仅 Apple，超时 5s |
| 全局快捷键 | 自定义唤出快捷键 | ⌥⌘T |
| 开机自启动 | 登录时自动启动 | 关闭 |

**引擎子页**：Settings Tab 内以导航方式进入"翻译引擎"子页（顶部显示返回按钮 + 标题）。子页内容：
- 已启用引擎列表（拖拽排序，右侧"移除"按钮；Apple 也可被移除）
- "添加引擎"按钮，点击后弹出菜单列出"尚未在列表中"的受支持引擎
- 请求超时时间数字输入（整数秒，默认 5，最小 1）

### 5. 菜单栏交互

- 左键点击菜单栏图标：弹出/关闭翻译窗口
- 右键点击菜单栏图标：显示上下文菜单，包含：设置、检查更新、关于、退出

## 八、异常处理与稳定性要求

1. **网络异常**：翻译请求失败时，在输出框显示友好的错误提示（如"网络连接失败，请检查网络"），不弹 alert
2. **翻译引擎不可用**：按用户设定的引擎列表从上到下尝试，任一引擎失败（错误或超时）即自动切到列表下一项；列表内所有引擎都失败后显示"全部引擎均失败"错误提示。不再做隐藏兜底，用户删除 Apple 就不会自动回退到 Apple
3. **Apple 翻译语言包未下载**：若 Apple 在列表中且本机未装语言包，视作本次 Apple 调用失败，继续尝试列表中的下一个引擎；若 Apple 是列表中唯一引擎或后续引擎也全部失败，则提示用户下载语言包
4. **Google 网页端点异常**：按普通 HTTP 错误处理并触发失败切换，不特殊化处理
5. **请求超时**：统一超时时间（默认 5 秒）触达即视为该引擎失败并切到下一个
6. **输入为空**：不触发翻译请求；同时把当前活动引擎重置为列表首位
7. **TTS 不支持该语言**：朗读按钮置灰或隐藏，不报错

## 九、隐私与安全要求

1. 翻译请求直接发往各翻译引擎官方端点，不经过任何中间服务器
2. 翻译历史仅存储在本地，不上传
3. App 不收集任何用户数据
4. 不存储任何 API Key、账号凭证或敏感信息（Google 引擎走公开网页端点，无凭证）

## 十、验收标准

1. App 启动后在菜单栏显示图标，无 Dock 图标
2. 点击菜单栏图标弹出翻译窗口，再次点击关闭
3. 全局快捷键（⌥⌘T）可唤出/关闭翻译窗口（弹出位置与点击菜单栏图标一致），支持自定义修改
4. 输入文本后 500ms 自动触发翻译，结果显示在输出框
5. 自动检测输入语言正确率 > 95%（依赖翻译引擎能力）
6. 输出语言"自动"模式：中文输入→英文输出，其他→中文输出，行为正确
7. Apple 翻译和 Google 翻译均开箱即用，无需配置
8. 首次安装引擎列表只含 Apple；可在设置的"翻译引擎"子页添加 Google（及未来的其他引擎）、支持拖拽排序、支持删除任意引擎（列表至少保留 1 个）
9. 引擎自动切换：翻译时从列表首位依次尝试，某引擎错误或超时（默认 5s，可配置，最小 1s）则切到下一个；全部失败显示错误，不再偷偷兜底 Apple
10. 输入框被清空后再次输入时，翻译从列表首位引擎重新开始
11. 复制按钮将文本复制到系统剪贴板
12. 朗读按钮使用系统 TTS 播放文本
13. 翻译历史正确记录并可查看、搜索、删除，上限 500 条
14. 点击历史条目可回填到翻译视图
15. 关闭窗口后内容保留 3 分钟，期满后自动清空并将下次打开的 Tab 重置为"翻译"
16. 开机自启动功能正常
17. 中英文切换后所有界面文本正确显示
18. 界面外观自动跟随系统浅色/深色模式切换
19. 所有用户设置在 App 重启后保持
20. Sparkle 自动更新：可检查更新、下载并安装
21. 常驻内存占用不超过 50MB

## 十一、总结

LingoBar 的核心竞争力是**原生体验 + 即时可用**：一个常驻菜单栏的轻量翻译工具，唤出即翻译，用完即关闭，不打断工作流。关键成功要素是翻译响应速度和 UI 的原生质感。
