# LingoBar

## 一、产品概述

LingoBar 是一款 macOS 菜单栏翻译工具。它常驻系统菜单栏，用户点击图标或按下全局快捷键即可唤出翻译窗口，快速完成多语言翻译。

产品定位：轻量、快速、原生。不做大而全的翻译平台，只做"点一下就能翻译"的效率工具。纯 Swift + SwiftUI 实现，无 WebView，遵循 Apple 设计语言，追求原生体验。

翻译能力由第三方翻译 API 提供（Google / 微软 / 百度 / 有道），用户自行配置 API Key。

---

## 二、产品目标

1. 从唤出翻译窗口到看到翻译结果，整个流程在 2 秒内完成（网络正常时）
2. 支持至少 20 种常见语言的互译（取各翻译引擎支持语言的交集）
3. 支持 4 个翻译引擎：Google 翻译、微软翻译、百度翻译、有道翻译
4. 内存占用常驻状态下不超过 50MB
5. UI 简洁美观，符合 macOS 原生设计语言

---

## 三、目标用户

1. **多语言工作者**：日常需要频繁查词、翻译短句的开发者、设计师、产品经理等，核心诉求是快速获取翻译结果，不打断工作流
2. **语言学习者**：需要随时查询词义、对比不同翻译引擎结果的用户，核心诉求是方便对照和朗读

## 五、核心用户流程

### 第一步：首次启动与配置

1. 用户首次打开 App，菜单栏出现 LingoBar 图标
2. 弹出引导提示，告知用户需要在设置中配置至少一个翻译引擎的 API Key
3. 用户进入设置页面，填入至少一个翻译引擎的 API Key
4. 配置完成后，App 进入可用状态

> **规则**：未配置任何有效 API Key 时，翻译功能不可用，输入框下方显示提示引导用户去设置

### 第二步：唤出翻译窗口

用户通过以下任一方式唤出翻译窗口：
- **点击菜单栏图标**：翻译窗口从菜单栏图标下方弹出
- **全局快捷键**：按下快捷键（默认 ⌥⌘T），翻译窗口出现在屏幕中央偏上位置

> **规则**：窗口已显示时再次触发，关闭窗口（toggle 行为）

### 第三步：输入与翻译

1. 翻译窗口弹出后，输入框自动获得焦点
2. 用户在输入框中输入文本
3. 实时翻译：用户停止输入约 500ms 后自动触发翻译
4. 翻译结果显示在输出框中

> **语言选择规则**：
> - 输入语言：默认"自动检测"，用户可手动切换为指定语言
> - 输出语言：选择器包含"自动"选项和所有具体语言。"自动"模式下启用智能切换（检测到中文→输出英文，其他语言→输出中文）；选择具体语言时固定为该语言
> - 输出语言默认值："自动"
> - 语言选择记忆：输入语言和输出语言的选择在 App 重启后保持

### 第四步：结果操作

用户可对输入文本和翻译结果执行以下操作：
- **复制**：点击复制按钮，将文本复制到剪贴板
- **朗读**：点击朗读按钮，使用系统 TTS 朗读文本

### 第五步：关闭窗口

用户通过以下任一方式关闭翻译窗口：
- 点击窗口外部区域
- 按 Esc 键
- 再次点击菜单栏图标
- 再次按下全局快捷键

> **内容保留规则**：关闭窗口后，输入内容和翻译结果保留一段时间（默认 1 分钟），在保留期内重新打开窗口可看到上次内容。保留时间可在设置中自定义，设为 0 表示关闭后立即清空

## 六、功能模块详述

### 1. 翻译窗口 UI

窗口采用上下分区布局，固定尺寸，不可调整大小。结构如下：

```
┌──────────────────────────┐
│  [翻译]  [历史]          │  ← 顶部 Tab 切换
├──────────────────────────┤
│ [自动检测 ▼]             │  ← 输入语言选择器
│                          │
│  输入文本区        [📋][🔊]│  ← 输入框 + 复制/朗读按钮
│                          │
├──────────────────────────┤
│ [自动 ▼]                 │  ← 输出语言选择器
│                          │
│  翻译结果区        [📋][🔊]│  ← 输出框 + 复制/朗读按钮
│                          │
├──────────────────────────┤
│  [Google] [MS] [Baidu]   │  ← 引擎 Segmented Control
└──────────────────────────┘
```

- **窗口类型**：点击菜单栏图标时以 Popover 形式从图标下方弹出；通过全局快捷键唤出时以浮动面板（NSPanel）形式出现在屏幕中央偏上位置
- **输入/输出框**：支持多行文本，内容超出时可滚动
- **历史视图**：切换到"历史"Tab 后，显示历史记录列表，顶部有搜索框，每条记录显示原文摘要、译文摘要、引擎图标和时间

### 2. 翻译引擎管理

App 支持 4 个翻译引擎，用户在设置中管理：

| 引擎 | API Key 配置项 | 备注 |
|------|---------------|------|
| Google 翻译 | API Key | Google Cloud Translation API |
| 微软翻译 | API Key + Region | Azure Cognitive Services Translator |
| 百度翻译 | App ID + 密钥 | 百度翻译开放平台 |
| 有道翻译 | App Key + 密钥 | 有道智云翻译 |

- 用户可配置多个引擎，在翻译窗口底部通过 Segmented Control 切换当前使用的引擎
- 仅显示已配置 Key 的引擎供切换
- 切换引擎后自动重新翻译当前内容

### 3. 翻译历史

- 保存翻译历史记录，包含：原文、译文、源语言、目标语言、使用的引擎、时间戳
- 翻译窗口顶部设 Tab 切换（翻译 / 历史），在窗口内查看历史
- 历史记录条数上限：500 条，超出后自动删除最早的记录
- 支持搜索历史记录（按原文和译文搜索）
- 支持删除单条 / 清空全部历史
- 点击历史记录条目可将内容回填到翻译视图

### 4. 设置

设置页面以独立窗口打开，包含以下配置项：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 翻译引擎 API Key | 各引擎的密钥配置 | 空 |
| 全局快捷键 | 自定义唤出快捷键 | ⌥⌘T |
| 开机自启动 | 登录时自动启动 | 关闭 |
| 外观模式 | 浅色 / 深色 / 跟随系统 | 跟随系统 |
| 语言 | App 界面语言 | 跟随系统 |
| 内容保留时间 | 关闭窗口后内容保留时长，0 表示立即清空 | 1 分钟 |
| 引擎失败策略 | 自动切换下一个可用引擎 / 仅提示错误 | 自动切换 |

### 5. 菜单栏交互

- 左键点击菜单栏图标：弹出/关闭翻译窗口
- 右键点击菜单栏图标：显示上下文菜单，包含：设置、检查更新、关于、退出

## 八、异常处理与稳定性要求

1. **网络异常**：翻译请求失败时，在输出框显示友好的错误提示（如"网络连接失败，请检查网络"），不弹 alert
2. **API Key 无效**：调用返回认证错误时，提示"API Key 无效，请检查设置"
3. **API 配额耗尽**：提示"API 调用次数已达上限"
4. **翻译引擎不可用**：默认自动切换到下一个已配置的可用引擎，全部失败后显示错误提示。用户可在设置中关闭自动切换，改为仅提示错误
5. **输入为空**：不触发翻译请求
6. **TTS 不支持该语言**：朗读按钮置灰或隐藏，不报错

---

## 九、隐私与安全要求

1. API Key 使用 macOS Keychain 存储，不明文保存
2. 翻译请求直接发往各翻译引擎官方 API，不经过任何中间服务器
3. 翻译历史仅存储在本地，不上传
4. App 不收集任何用户数据

---

## 十、验收标准

1. App 启动后在菜单栏显示图标，无 Dock 图标
2. 点击菜单栏图标弹出翻译窗口，再次点击关闭
3. 全局快捷键（⌥⌘T）可唤出/关闭翻译窗口，支持自定义修改
4. 输入文本后 500ms 自动触发翻译，结果显示在输出框
5. 自动检测输入语言正确率 > 95%（依赖翻译引擎能力）
6. 输出语言"自动"模式：中文输入→英文输出，其他→中文输出，行为正确
7. 4 个翻译引擎均可正常配置和使用
8. 翻译引擎切换后自动重新翻译当前内容
9. 引擎失败自动切换：当前引擎失败后自动尝试下一个可用引擎
10. 复制按钮将文本复制到系统剪贴板
11. 朗读按钮使用系统 TTS 播放文本
12. 翻译历史正确记录并可查看、搜索、删除，上限 500 条
13. 点击历史条目可回填到翻译视图
14. 关闭窗口后内容按配置的保留时间保持，过期后自动清空
15. 开机自启动功能正常
16. 中英文切换后所有界面文本正确显示
17. 浅色/深色/跟随系统三种模式切换正常
18. 所有用户设置在 App 重启后保持
19. API Key 存储在 Keychain 中，非明文
20. Sparkle 自动更新：可检查更新、下载并安装
21. 常驻内存占用不超过 50MB

---

## 十一、总结

LingoBar 的核心竞争力是**原生体验 + 即时可用**：一个常驻菜单栏的轻量翻译工具，唤出即翻译，用完即关闭，不打断工作流。关键成功要素是翻译响应速度和 UI 的原生质感。
